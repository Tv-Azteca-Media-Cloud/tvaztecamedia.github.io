2.1 ASM on Platform — Roles y Terminología

Advertising Sales Management define dos grupos de roles con responsabilidades diferenciadas sobre la plataforma.

Roles de Implementación
  • Admin: configura objetos, picklists, permission sets, page layouts y activaciones en Setup
  • System Integrator: desarrolla y mantiene integraciones técnicas con Ad Servers externos (GAM)
Roles de Operación
  • Ad Sales Admin: gestión completa del proceso de venta publicitaria
  • Media Planner: creación y configuración de Media Plans y placements
  • Sales Rep: gestión de oportunidades y propuestas comerciales
  • Ad Operations Specialist: ejecución y seguimiento de campañas

2.3 Habilitar funciones de venta publicitaria

Pasos de habilitación en Salesforce Setup requeridos antes de operar cualquier flujo de Media Plan.

  1. Media Cloud Configuration
    Activar la configuración de disponibilidad de anuncios por defecto. Habilita el botón "Create Media Plan" en Opportunity.
  2. Context Service
    Activar Context Service para MediaPlanTransactionContext — requerido por el motor de cotización y pricing en tiempo real.
  3. Revenue Cloud Features
    Habilitar funciones de Revenue Cloud: quotes, orders, Salesforce Pricing, Instant Pricing, Runtime Product Configuration y Business Rules Engine.
2.4 Media Cloud Configuration
// Ruta de activación Setup → Media Cloud Configuration → Activar

Efecto de la activación: carga el paquete de metadata JSON base que incluye páginas, componentes, pestañas y modales del plan de medios. Este proceso instala los elementos de interfaz necesarios para operar dentro de las Opportunities.

Crítico Sin habilitar Media Cloud Configuration, el botón "Create Media Plan" NO aparece en la Opportunity. Esta configuración es el punto de partida absoluto de la implementación.
2.5 Revenue Cloud Settings

Los siguientes 5 pasos deben ejecutarse en orden estricto. El primer paso es obligatorio y desbloquea los pasos subsecuentes.

  1. Enable Revenue Cloud Features
    Setup → Revenue Settings
    Primer paso obligatorio. Activa el motor Revenue Lifecycle Management y expone todos los settings posteriores.
  2. Salesforce Pricing
    Habilita el Price Waterfall completo — permite que los precios se calculen en cascada según las reglas definidas en la Pricing Procedure.
  3. Instant Pricing
    Recalcula montos en la UI sin necesidad de guardar el registro. Indispensable para la experiencia fluida en el Media Planning Grid.
  4. Configure Products at Runtime
    Habilita la visualización de los flujos UI de configuración de productos en la línea de cotización (Line Item Configuration panel).
  5. Business Rules Engine (BRE)
    Activa las tablas de decisión y Expression Sets utilizados en reglas de negocio personalizadas de TV Azteca.
Quote Settings
  • Enable Quotes — habilita el objeto Quote en el org
  • Crear cotizaciones sin oportunidad relacionada — permite flexibilidad operativa
  • Agregar Quotes a Page Layout de Opportunity — visibilidad del panel de cotizaciones
Order Settings
  • Enable Orders — habilita el objeto Order
  • Enable Negative Quantity — necesario para ajustes y créditos
  • Enable Zero Quantity — permite líneas de cortesía
  • Enable Enhanced Commerce Orders — ver nota crítica
Crítico — Enhanced Commerce Orders Sin este setting habilitado, la línea de cotización no puede convertirse a Order correctamente. Su activación expone el campo Total Line Amount y hace opcional el campo Unit Price, ambos requeridos en el flujo de media plan.

§ 4

PCM Inicial — Product Catalog Management

Configuración del catálogo de productos digitales. El PCM es la capa de configuración que define qué se vende, cómo se presenta y cómo se tarifica en el flujo de Media Plan.

3.1 Estructura del Catálogo
CampoValor
Catalog NameCatálogo Digital TV Azteca
CategoríasStreaming, Digital Display, Patrocinio, Servicios
Scope inicialBase catalog — productos de lanzamiento
Alcance Sprint 1 El catálogo inicial cubre los productos digitales necesarios para activar el primer flujo de Create Media Plan. Los productos adicionales se incorporan en sprints posteriores conforme se valida el modelo de precios.
3.2 Modelo de 3 Capas

El PCM opera con un modelo de tres capas. Las tres deben estar correctamente configuradas para que un producto sea visible, cotizable y orquestable en el flujo de ventas.

Capa 1
PCM Comercial
¿QUÉ se vende?
Product Catalog ProductCategory Product2 ProductSellingModel PricebookEntry ProductAttributeDefinition
Selling Models:
• Term-Based (CPM) — vigencia definida, prorrateo automático mid-cycle
• One-Time (FIJO) — pago único, sin prorrateo
Capa 2
Ad Specification
¿DÓNDE y CÓMO?
MediaChannel AdSpaceSpecification AdCreativeSizeType AdSpaceSpecProduct
Nomenclatura: [Canal] - [AdSpaceType] - [Dispositivo]
Ejemplo: "TV Azteca Uno - Pre-roll - Desktop"
Crítico — AdServerId El campo AdServerId es OBLIGATORIO en AdSpaceSpecification. Sin él se produce un error de runtime al cargar el Media Plan — el panel no se renderiza.
Capa 3
Targeting
¿A QUIÉN?
AdTargetCategory AdTargetCategorySegment AdTargetSegmentValue AdAvailableTargetSegment AdAvailableTargetSegmentValue
Cascading: Campo DependantSegmentValueId en AdTargetSegmentValue permite valores de segmento dependientes (cascada).

Regla de visibilidad: Un producto debe cumplir las 3 capas completas para ser visible, cotizable y orquestable.
3.3 Campos Requeridos en Product2

Todos los campos listados deben estar correctamente configurados. Un campo faltante o con valor incorrecto genera fallos silenciosos en la UI o errores en el motor de precios.

Campo Valor requerido Impacto si falta / es incorrecto
IsActive true No aparece en Browse Product Catalog
ConfigureDuringSale Allowed No configurable en Line Item Configuration panel
CanUseRevenueSchedule true Sin prorrateo automático en Term-Based
ProductPurpose Sell No reconocido por motor de precios
ProductSellingModel Term-Based / One-Time Sin modelo de venta no se asocia PricebookEntry
3.4 Pricing Models
Modelo Fórmula Selling Model
CPM Precio × (Impresiones / 1,000) Term-Based
CPC Precio × Número de clics Term-Based
FIJO Tarifa plana (flat fee) One-Time
§ 5

Flujo Base Media Plan

Configuración del flujo comercial completo desde la oportunidad hasta la creación de la orden firme. Cubre pricing procedure, transaction types y el proceso Quote to Order.

5.1 Pricing Procedure
AtributoValor
NameTVAM Ad Sales Pricing Procedure
ContextTVAM_MediaPlanTransactionCtx
ProrationProration Settings activas
Proration Settings Configuradas para habilitar el prorrateo mid-cycle automático en productos Term-Based. Cuando se activa o pausa una línea en mitad de un ciclo, el sistema calcula el monto proporcional sin intervención manual.
5.2 Sale Transaction Types
Media Plan Digital
Propuesta estándar de ventas digitales. Se usa para todos los Media Plans generados desde una Opportunity activa.
Media Plan Template
Plantilla reutilizable para planificadores. Permite crear estructuras base de plan que se clonan para múltiples clientes.
5.3 Quote to Order — Flujo Completo
1
Oportunidad (RFP)
El vendedor abre la Opportunity y hace clic en el botón Create Media Plan. Este botón es habilitado por la Media Cloud Configuration.
2
Modal Nuevo Plan
Se ingresan: nombre del plan, Budget (ej: $1,000,000), Media Type → El sistema genera automáticamente un registro Quote ligado a la Opportunity.
3
Quote Record Page — Media Planning Grid
La página del Quote carga el Media Planning Grid con los Ad Placements. Este es el espacio de trabajo principal del planificador.
4
Add Placements
El planificador selecciona el Standard Price Book y el tipo Digital para acceder al catálogo de productos PCM.
5
Browse Products
Se navega el catálogo PCM organizado por categorías. Al seleccionar un producto y guardar → el sistema genera un Ad Quote Line.
6
Line Item Configuration
Panel de 3 pestañas:
• Line Item — modelo de precio (CPC/CPM), importes, fechas
• Delivery — frecuencia, días, distribución temporal
• Targeting — segmentos de audiencia
Instant Pricing recalcula en background sin guardar.
7
Create Order
Invoca el flujo TVAM_ProcesarOrden. La Quote se convierte en Order firme e inicia el aprovisionamiento DRO. Requiere que la Quote esté en etapa Approved.
